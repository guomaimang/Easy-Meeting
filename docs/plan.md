# 产品计划

## 定位

Easy Meeting 的目标是完整 macOS 本地会议助手产品，不是一次性验证项目。

## 目标

第一阶段交付 Easy Meeting 的核心会议闭环：

1. 在 macOS 桌面显示透明悬浮字幕窗。
2. 用户可以选择麦克风。
3. 采集麦克风音频并发送到火山引擎语音服务。
4. 实时显示原文和译文。
5. 本地保存录音、转录、翻译和历史记录。
6. 导出录音和转录文件。

## 成功标准

- App 能启动为菜单栏应用。
- 悬浮窗可显示、隐藏、调节透明度，并具备继续扩展拖动和锁定能力的结构。
- 麦克风权限请求正常。
- 至少能使用默认输入设备录音。
- 录音开始时创建本地会议目录和 `metadata.json`。
- 英文转中文模式能显示实时字幕。
- 火山模式能按 AST 协议发送音频并解析字幕事件。
- 会议结束后能在本地历史记录中看到本次会议。
- 能导出 `.m4a` 和 `.txt`。
- 菜单栏可以查看最近会议。
- 错误状态能在悬浮窗和菜单入口得到可理解反馈。
- 代码结构允许后续继续接入快捷键、字幕样式、导出位置选择、诊断日志和更多语音服务。

## 完整 APP 验收路线

### 真实语音闭环

- 设置窗口保存火山 API Key，并展示固定 Resource ID。
- 设置窗口可以检查本地 helper 和火山配置是否齐全。
- 录音开始时建立 AST WebSocket 连接。
- 通过 Go helper 复用官方 AST 客户端协议实现。
- 音频帧统一为 PCM 16kHz、单声道、16bit 后再发送。
- 发送 `StartSession`、`TaskRequest` 音频帧和 `FinishSession`。
- 解析源语言字幕和目标语言字幕事件。
- 字幕写入悬浮窗、SQLite 和会议导出文件。

### 会议工作流

- 会前可以选择输入设备、翻译模式和语音服务配置。
- 首次启动或配置不完整时自动打开设置窗口，并在悬浮窗提示入口位置。
- 会中可以开始、停止录音，并看到服务连接状态和错误反馈。
- 会后可以从最近会议查看和导出音频、转录、字幕和 JSON。

### 产品化体验

- 悬浮窗支持拖动、缩放、锁定、位置记忆、字号和透明度设置。
- 设置页提供配置校验和本地数据管理入口。
- 导出支持选择目录、批量导出和失败重试。
- 日志区分开发 DEBUG 和生产 INFO，关键链路埋点可诊断。
- 屏幕共享边界有实测记录和用户可理解说明。

## 屏幕共享验证

需要手工验证以下场景：

| 软件 | 共享窗口 | 共享整个屏幕 |
|---|---|---|
| Zoom | 需要测 | 需要测 |
| Google Meet | 需要测 | 需要测 |
| Microsoft Teams | 需要测 | 需要测 |
| 腾讯会议 | 需要测 | 需要测 |
| 飞书会议 | 需要测 | 需要测 |

预期：

- 共享单个窗口时，悬浮窗大概率不会被共享。
- 共享整个屏幕时，悬浮窗可能被共享。
- 产品只做 best-effort 隐私悬浮层，不使用私有 API。

## 第一阶段模块

- App shell：菜单栏入口和主状态管理。
- Overlay：透明悬浮字幕窗。
- Audio：麦克风枚举、选择、采集、录音。
- Storage：SQLite 和会议文件目录。
- Speech：火山实时语音服务封装。
- History：菜单栏最近会议列表。
- Export：录音和文本导出。

## 产品化待办

- Go helper 的火山 AST 响应事件分组、字幕配对和错误恢复。
- 音频采集管道拆分，让录音和实时语音服务共享同一份采集帧。
- 删除或隔离所有用户可见的模拟服务入口，内部测试替身不得进入默认产品路径。
- 会议元数据补齐模式、源语言、目标语言和服务商信息。
- 悬浮窗锁定、位置记忆、字号和显示密度设置。
- 导出位置选择、批量导出和错误提示。
- 开发环境细粒度日志和生产环境 INFO 级别日志。
- 屏幕共享实测记录和用户可理解的隐私边界说明。

## 开发顺序

1. 搭建 macOS 原生工程。
2. 实现菜单栏和悬浮窗。
3. 实现麦克风权限和输入设备选择。
4. 实现音频采集、录音和会议目录。
5. 实现 SQLite 存储和会议目录。
6. 接入火山实时语音服务，跑通字幕和分段存储。
7. 用内部测试替身补齐自动化验证，不暴露为用户可选服务。
8. 实现历史记录和导出。
9. 做屏幕共享实测记录。
