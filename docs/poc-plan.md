# POC 计划

## 目标

验证 Easy Meeting 的核心闭环：

1. 在 macOS 桌面显示透明悬浮字幕窗。
2. 用户可以选择麦克风。
3. 采集麦克风音频并发送到火山引擎语音服务。
4. 实时显示原文和译文。
5. 本地保存录音、转录、翻译和历史记录。
6. 导出录音和转录文件。

## 成功标准

- App 能启动为菜单栏应用。
- 悬浮窗可显示、隐藏、拖动、调节透明度。
- 麦克风权限请求正常。
- 至少能使用默认输入设备录音。
- 录音开始时创建本地会议目录和 `metadata.json`。
- 英文转中文模式能显示实时字幕。
- 会议结束后能在本地历史记录中看到本次会议。
- 能导出 `.m4a` 和 `.txt`。
- Mock 实时语音事件能写入 SQLite 和 `transcript.txt`。
- 菜单栏可以查看最近会议。

## 屏幕共享验证

需要手工验证以下场景：

| 软件 | 共享窗口 | 共享整个屏幕 |
|---|---|---|
| Zoom | 需要测 | 需要测 |
| Google Meet | 需要测 | 需要测 |
| Microsoft Teams | 需要测 | 需要测 |
| 腾讯会议 | 需要测 | 需要测 |
| 飞书会议 | 需要测 | 需要测 |

预期：

- 共享单个窗口时，悬浮窗大概率不会被共享。
- 共享整个屏幕时，悬浮窗可能被共享。
- POC 只做 best-effort 隐私悬浮层，不使用私有 API。

## 第一阶段模块

- App shell：菜单栏入口和主状态管理。
- Overlay：透明悬浮字幕窗。
- Audio：麦克风枚举、选择、采集、录音。
- Storage：SQLite 和会议文件目录。
- Speech：火山实时语音服务封装。
- History：菜单栏最近会议列表。
- Export：录音和文本导出。

## 开发顺序

1. 搭建 macOS 原生工程。
2. 实现菜单栏和悬浮窗。
3. 实现麦克风权限和输入设备选择。
4. 实现音频采集、录音和会议目录。
5. 实现 SQLite 存储和会议目录。
6. 接入 Mock 实时语音事件，跑通字幕和分段存储。
7. 接入火山实时语音服务。
8. 实现历史记录和导出。
9. 做屏幕共享实测记录。
